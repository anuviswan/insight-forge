# Purpose
Conduct comprehensive research and generate an accurate and structured research artifact for downstream blog-writing skills.

This skill is responsible for gathering information, identifying key concepts, analyzing trade-offs, and producing a research package that can be consumed by domain experts, outline generation, fact checking, and content creation.


# When to use

- Researching a new technical topic
- Validating claims
- Collecting references and sources
- Comparing technologies, frameworks, or approaches
- Finding best practices
- Understanding architecture and patterns
- Gathering Industrial standards, statistics, benchmarks and performance data


# Instructions

- Clarify requirements. If requiremetns are not clear, ask question. NEVER ASSUME
- Primary Sources 
  - Official documentation
  - Academic Papers
  - Specifications and standards
- Secondary Sources
  - Engineering Blogs from recognized companies
  - Conference Talks
- Terinary Scources
  - Engineering blogs from reputed domain experts
  - Community Contents (Stackoverflow, Microsoft Forums)
- Verify important claims
- Never present assumptions as facts
- For evolving technologies, prefer published information within last 24 months
- Check for depreciations and breaking changes
- Check current recommendations

# Research Process

 - 1. Clarify - Understand the research objective clearly, ASK questions when required. NEVER ASSUME
 - 2. Collect Information - Search broadly and then refine
 - 3. Validate Information - Validate facts across multiple sources
 - 4. Organize Findings - Organize the findings

# Output

 - Key Findings
 - Best Practices and recommendations
 - Code Samples
 - Common Pitfalls
 - References
